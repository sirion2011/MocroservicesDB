ЗАДАЧА: РАЗРАБОТКА ГРАФОВОЙ БАЗЫ ДАННЫХ ДЛЯ МИКРОСЕРВИСНОЙ АРХИТЕКТУРЫ

Необходимо спроектировать и реализовать графовую базу данных на платформе MS SQL Server для моделирования взаимодействия микросервисов в распределенной системе.

СУЩНОСТИ (УЗЛЫ / NODES):
1. Service (микросервис) — содержит идентификатор ServiceID и название ServiceName.
   Примеры экземпляров: GatewayService, UserService, OrderService, PaymentService.
   
2. API (интерфейс программирования приложения) — содержит идентификатор APIID, название APIName и параметры APIParameters.
   Каждый микросервис предоставляет набор API-эндпоинтов (например, /user/profile, /order/create, /payment/charge).

3. Database (база данных) — содержит идентификатор DatabaseID и тип DBType.
   Микросервисы используют различные типы баз данных: PostgreSQL (основная БД) и Redis (кэш).

СВЯЗИ (РЁБРА / EDGES):
1. Expose (предоставляет) — направленная связь от Service к API.
   Атрибут ребра: Tariff (тариф в денежных единицах) — стоимость использования данного API.
   
2. Calls (вызывает) — направленная связь от Service к другому Service.
   Атрибут ребра: Status (статус вызова: 'success', 'error', 'timeout').
   Данная связь моделирует межсервисное взаимодействие (синхронные вызовы).
   
3. UsesDatabase (использует БД) — направленная связь от Service к Database.
   Атрибут ребра: AccessType (тип доступа: 'read' — только чтение, 'both' — чтение и запись).

ОГРАНИЧЕНИЯ (CONNECTION CONSTRAINT):
- Связь Expose может соединять только вершины типа Service и API.
- Связь Calls может соединять только две вершины типа Service.
- Связь UsesDatabase может соединять только вершины типа Service и Database.

ЗАПОЛНЕНИЕ ДАННЫМИ:
- Таблица Service: 4 записи (4 микросервиса).
- Таблица API: 14 записей (все эндпоинты всех сервисов).
- Таблица Database: 8 записей (по 2 БД на каждый сервис — PostgreSQL и Redis).
- Таблица Expose: 14 записей (каждый API привязан к своему сервису с указанием тарифа).
- Таблица Calls: 6 записей (связи вызовов между микросервисами с указанием статуса).
- Таблица UsesDatabase: 8 записей (каждый сервис использует свои БД с указанием типа доступа).

ЦЕЛИ РАБОТЫ:
1. Реализовать графовую базу данных с использованием синтаксиса AS NODE и AS EDGE.
2. Написать 5 различных запросов с использованием функции MATCH для анализа цепочек взаимодействия (например, найти сервисы, которые используют PostgreSQL и вызывают другой сервис с ошибкой).
3. Написать 2 запроса с использованием SHORTEST_PATH для поиска кратчайших путей между сервисами.
4. Визуализировать полученный граф в Power BI с помощью Force-Directed Graph.
5. Создать диаграммы графа в diagrams.net.
